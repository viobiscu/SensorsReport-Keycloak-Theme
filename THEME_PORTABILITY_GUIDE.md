# Keycloak Theme Portability Guide

## Problem
When moving the SensorsReport theme between different Keycloak instances, hardcoded resource URLs with cache-busting identifiers (like `/resources/3pw76/`) would break because each Keycloak instance generates its own unique identifier.

## Solution
We've implemented dynamic resource path resolution that automatically adapts to any Keycloak instance.

## Changes Made

### 1. FreeMarker Templates Updated
All templates now pass the dynamic resource path to JavaScript:

**login/template.ftl:**
```html
<script>
    window.KEYCLOAK_RESOURCE_PATH = "${url.resourcesPath}";
</script>
```

**account/index.ftl:**
```html
<script>
    window.KEYCLOAK_RESOURCE_PATH = "${(url.resourcesPath)!'/resources'}";
</script>
```

**admin/template.ftl:**
```html
<script>
    window.KEYCLOAK_RESOURCE_PATH = "${resourcesPath}";
</script>
```

### 2. JavaScript Updated
The `theme.js` file now uses dynamic paths instead of hardcoded ones:

**Before:**
```javascript
const backgrounds = [
  '/resources/3pw76/login/SensorsReport/img/pharmaceutical-warehouse.jpg',
  // ... more hardcoded paths
];
```

**After:**
```javascript
function getBackgroundPaths() {
  const basePath = window.KEYCLOAK_RESOURCE_PATH || '/resources';
  const images = [
    'pharmaceutical-warehouse.jpg',
    'pharmaceutical-warehouse-02.jpg',
    // ... image names only
  ];
  
  return images.map(image => `${basePath}/img/${image}`);
}
```

## How It Works

1. **Keycloak provides dynamic paths**: Each template type has access to resource path variables:
   - Login: `url.resourcesPath`
   - Account: `url.resourcesPath` 
   - Admin: `resourcesPath`

2. **Template injects the path**: The FreeMarker template renders the actual path and makes it available to JavaScript via `window.KEYCLOAK_RESOURCE_PATH`

3. **JavaScript uses dynamic path**: Instead of hardcoded URLs, JavaScript builds paths dynamically using the provided base path

## Benefits

✅ **Portable**: Theme works on any Keycloak instance without modification
✅ **Cache-friendly**: Automatically uses Keycloak's cache-busting identifiers
✅ **Maintainable**: No need to update hardcoded paths when moving between environments
✅ **Future-proof**: Adapts to Keycloak's internal URL changes automatically

## Deployment Instructions

When deploying this theme to a new Keycloak instance:

1. Copy the entire `SensorsReport` theme folder to `themes/` directory
2. Restart Keycloak or run `kc.sh build`
3. No additional configuration needed - the theme will automatically adapt to the new instance

## Testing

To verify the solution works:

1. Access the login page and check browser dev tools
2. Look for the `window.KEYCLOAK_RESOURCE_PATH` variable in the console
3. Verify background images load correctly from the dynamic path
4. Test on multiple Keycloak instances to confirm portability

## Technical Notes

- The cache-busting identifier (like `3pw76`) is generated by Keycloak automatically
- It changes when themes are updated or Keycloak restarts
- The identifier ensures browsers don't serve cached versions of old resources
- You should never try to control or hardcode this identifier
